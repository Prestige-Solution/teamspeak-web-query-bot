# Contributing

Thank you for your interest in contributing to this project. Contributions of all kinds are welcome—bug reports, feature requests, documentation improvements, and code.<br>
Please take a moment to review these guidelines to help keep the contribution process smooth and effective for everyone.

---

## Code of Conduct

This project follows a Code of Conduct. By participating, you agree to uphold a respectful, inclusive, and constructive environment.

---

## How to Contribute

### 🐛 Reporting Bugs

If you find a bug:

1. Check the existing issues to ensure it hasn’t already been reported.
2. Open a new issue and include:
    - What happened (actual behavior)
    - What you expected to happen
    - Steps to reproduce
    - Your environment (OS, version, browser/runtime, etc.)
    - Logs/screenshots if relevant

---

### 💡 Suggesting Features

Feature ideas are welcome.<br> 
When proposing a feature, please include:

- The problem you’re trying to solve
- Your proposed solution
- Alternatives you’ve considered (optional)
- Any additional context or examples

---

### 🔧 Contributing Code

1. Fork the repository
2. Create a new branch:
    ```bash
    git checkout -b feature/my-feature
    ```
3. Make your changes
4. Make sure that:
   - the project builds/runs successfully 
   - tests pass (if applicable)
   - code is formatted consistently
5. Commit your changes:
   ```
   git commit -m "feat: short description of the change"
   ```
6. Push your branch:
   ```bash
   git push origin feature/my-feature
   ```
7. Open a Pull Request

---

### Pull Request Guidelines

- Keep Pull Requests small and focused
- Explain what you changed and why
- Reference related issues (e.g., Closes #12)
- Be responsive to feedback and requested changes

---

### Coding Guidelines

- Write clear, readable code
- Follow existing project conventions and style
- Comment complex logic where helpful
- Avoid unnecessary dependencies

---

### Questions?

If you’re unsure about anything, feel free to:
- open an issue, or
- start a discussion

Thanks again for contributing!
